# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

FastAPI (backend) + React/TypeScript (frontend) demo application showcasing the SDS Manager API. It allows users to search a database of 13 million+ Safety Data Sheets (SDS), retrieve extracted data, upload SDS PDFs for extraction, and access safety information summaries.

- **Backend entry point:** `backend/app/main.py`
- **Backend settings:** `backend/app/core/config.py`
- **Backend API routes:** `backend/app/api/sds.py` (prefix: `/sds`)
- **Backend services:** `backend/app/services/sds_service.py`
- **Frontend entry:** `frontend/src/index.tsx`
- **Frontend routes:** `frontend/src/routes.jsx`
- **Frontend API client:** `frontend/src/api/index.js`

## Core Standards

- Keep all generated code production-ready and strongly typed where applicable.
- Prefer small focused files and functions over large mixed-responsibility modules.
- Match existing project conventions before introducing new patterns.
- Never ship placeholder TODO logic in production paths.
- No `print()` in backend code — use Python's `logging` module.

## Architecture Conventions

- All backend routes are async; use `await` throughout the service and client layers.
- Pydantic schemas live in `backend/app/schemas/` — use them for all request/response typing.
- Business logic lives in `backend/app/services/`, not in route handlers.
- External HTTP calls go through `SDSAPIClient` in `backend/app/clients/`.
- Rate limiting is applied per-route via `@limiter.limit("5/minute")` — do not bypass it.
- All SDS IDs passed externally are Fernet-encrypted using `SECRET_KEY`; encrypt/decrypt in `SDSService`.
- Frontend API calls go through the shared axios instance in `frontend/src/api/index.js`.
- Frontend form state is managed with Formik + Yup — do not introduce a separate form library.

## Security & Reliability Checklist

- Validate all user-controlled inputs at the route level via Pydantic schemas.
- API key is resolved from `X-SDS-SEARCH-ACCESS-API-KEY` header or environment fallback — never hardcode it.
- Avoid leaking secrets (API keys, SECRET_KEY) in logs or error responses.
- Fail safely: raise the correct custom exception class and let the route handler map it to HTTP status codes.
- Add `Retry-After` headers on `429` responses when the upstream provides a retry window.

## AI Generation Behavior

- Generate minimal diffs rather than full rewrites.
- Explain risky changes before applying them.

## Common Commands

### Backend

All backend commands run from `backend/`:

```bash
# Activate virtual environment
source venv/bin/activate

# Install dependencies
pip install -r requirements.txt

# Run dev server (hot reload)
uvicorn app.main:app --reload --port 8003

# API docs (auto-generated by FastAPI)
open http://127.0.0.1:8003/docs

# Run via Docker
docker-compose up
```

### Backend Linting & Formatting

Pre-commit runs automatically on `git commit`. To run manually:

```bash
pre-commit run --all-files

# Or individually
black backend/ --line-length 79
isort backend/
flake8 backend/ --max-line-length=79 --ignore=E203,W503,E501,E402,B008
```

### Frontend

All frontend commands run from `frontend/`:

```bash
# Install dependencies
npm install

# Start dev server (http://localhost:3000)
npm start

# Production build
npm run build

# Run tests
npm test
```

## Architecture

### Backend Structure

```
backend/
  app/
    api/
      sds.py            # All /sds/* route handlers
      dependencies.py   # FastAPI Depends() — API key + SDSService injection
    clients/
      sds_api_client.py # Async httpx client pool for external SDS Manager API
    core/
      config.py         # Pydantic BaseSettings (loads from .env)
    exceptions/
      sds_api.py        # Custom exception classes
    schemas/
      sds.py            # Pydantic request/response models
    services/
      sds_service.py    # Business logic — wraps SDSAPIClient, encrypts IDs
    throttling.py       # slowapi CustomLimiter (bypasses limit for API key holders)
    main.py             # FastAPI app, middleware, exception handlers, lifespan
  pyproject.toml        # Black + isort configuration
  requirements.txt      # Python dependencies
  .pre-commit-config.yaml
  Dockerfile
  docker-compose.yml
```

### API Endpoints

All endpoints under `/sds` prefix. All `POST` routes are rate-limited to `5/minute` per IP (bypassed if `X-SDS-SEARCH-ACCESS-API-KEY` header is present).

| Method | Path | Description |
|--------|------|-------------|
| `POST` | `/sds/search/` | Search SDS by product name, CAS, supplier, etc. |
| `POST` | `/sds/details/` | Get full extracted data for an SDS by ID or PDF MD5 |
| `POST` | `/sds/multipleDetails/` | Batch get SDS details |
| `POST` | `/sds/get-dif-language-versions/` | List language variants of an SDS |
| `POST` | `/sds/newRevisionInfo/` | Check for a newer revision of an SDS |
| `POST` | `/sds/multipleNewRevisionInfo/` | Batch revision check |
| `POST` | `/sds/upload/` | Upload SDS PDF for extraction (multipart) |
| `GET`  | `/sds/getExtractionStatus/` | Poll extraction status by `request_id` |
| `POST` | `/sds/safetyInformationSummary/` | Download safety summary PDF |

### Key Schemas (Pydantic)

- `BaseSDSSchema` — base model with Fernet ID encryption validator
- `ListSDSSchema` — minimal SDS info (search result rows)
- `SDSDetailsSchema` — full extracted SDS data
- `SearchSDSFilesBodySchema` — search query with date range validation
- `SDSDetailsBodySchema` — lookup by `sds_id` or `pdf_md5`

### Key Exception Classes

Defined in `backend/app/exceptions/sds_api.py`. Route handlers catch these and map to HTTP status codes:

| Exception | HTTP Status |
|-----------|-------------|
| `SDSNotFoundException` / `SDSNotFoundError` | 404 |
| `SDSBadRequestException` / `SDSAPIParamsRequired` | 400 |
| `SDSAPIRequestNotAuthorized` | 401 |
| `SDSAPIRateLimitError` | 429 |
| `SDSAPIInternalError` | 500 |

### Environment Detection (Frontend)

The `getEnv()` utility maps hostname to environment:

| Environment | Backend URL |
|-------------|-------------|
| `production` | `https://api.sdsmanager.com` |
| `rc` | `https://rc-api.sdsmanager.com` |
| `staging` | `https://staging-demo.sdsmanager.com` |
| `local` | `http://127.0.0.1:8003` |

### Configuration (Backend `.env`)

Required environment variables:

```env
CORS_ORIGINS=["http://localhost:3000"]
CORS_ALLOW_CREDENTIALS=true
CORS_ALLOW_METHODS=["*"]
CORS_ALLOW_HEADERS=["*"]
SDS_API_URL=<external SDS Manager API base URL>
SECRET_KEY=<base64 Fernet key for ID encryption>

# Optional
SDS_API_KEY=<fallback API key for env-origin requests>
SDS_API_TIMEOUT=120
VALUE_LIMIT=100
SDS_MAX_FILE_SIZE=5242880
REDIS_HOST=
REDIS_PORT=
REDIS_DB=
REDIS_PASSWORD=
```
